 Roadmap: What's Next for "ConU Planner 2.0"?
If you decide to open this codebase again for a major update, here are three high-impact directions that would take this from a "tool" to an "essential platform":

1. The Social Synchronicity (Community Layer)
Currently, the planner is a solo experience. The next leap is Social Discovery:

Shared Sequences: Allow students to "follow" a friend's plan or see the most "upvoted" elective sequences.
Classmate Discovery: A "Who's in my class?" widget on course cards that shows other users who have added that course to their sequence.
Peer Reviews 2.0: Integrating real student sentiment directly into the "Glass Island" modal for courses.
2. Clara 2.0: The Proactive Advisor
Transition Clara from a "helper" to a "strategist":

Automated Degree Audit: Clara could scan a student's sequence and proactively say: "Warning: You're missing a 400-level elective for graduation in Fall 2026."
GPA Simulation: AI-driven "What-if" scenarios: "If you get a B in COMP 352, your graduating GPA will be 3.8."
3. Real-Time Data Ecosystem
Direct API Integration: If possible, pulling real-time "Seat Availability" from Concordia’s actual schedule.
Course "Survival Guides": Adding a community-driven "Resource Hub" to each course card where students can link to public GitHub notes or study Discord servers.
4. Mobile-Native "Companion"
While the web app is responsive, a specialized React Native companion app focused on "Today's Schedule" and "Task Tracking" would complete the ecosystem.

New Feature: University Seat Finder (Real-time Registration Alerts)
Objective:
Notify students via SMS/Email the moment a seat opens up in a course section (or a waitlist spot becomes available).

Technical Strategy (Concordia University Example):

Public Data Source: Use the Concordia Public Class Search. This portal is public and provides live Capacity, Enrollment Total, and Waitlist counts without requiring any student login (NetID).
The "Extractor" (Scraper):
Build a bot using Puppeteer or Playwright (Node.js).
The bot will automatically navigate the public search, input the course code (e.g., COMP 248), for the specific term.
It will "click" on the class link to expand details and scrape the enrollment numbers.
Monitoring Logic:
Run the scraper on a loop (every 1–5 minutes) for high-demand courses.
Compare Enrollment Total against Class Capacity. If Enrollment < Capacity, trigger the alert.
Notification Stack:
SMS: Twilio API.
Email: Resend or SendGrid.
Monetization Idea: Once traffic picks up, introduce a "Pay-per-alert" model ($1.50 - $2.00 per course) using Stripe.
Action Items for Next Phase:

Set up a basic Node.js scraper script to test the Concordia Guest Portal data extraction.
Design a "Cyber-Glass" UI component for users to "Watch" a course.
Set up a MongoDB database to track active watches and course statuses.
